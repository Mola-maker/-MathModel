**改动说明（仅 `ui/server.py`，无其他文件）：**

| 新增内容 | 作用 |
|---------|------|
| `_sandbox_proc` 全局变量 | 持有沙箱 `docker exec` 的 `Popen` 句柄 |
| `_sandbox_output_buf` | 滚动缓存最近 200 行输出，供 assistant 直接读取 |
| `_start_sandbox_proc()` | docker cp 脚本 → Popen 启动 docker exec（非阻塞） |
| `sandbox_run` action | 运行 `vol/scripts/` 下指定 `.py` 脚本 |
| `sandbox_exec` action | 执行任意 Python 代码（写入 `_assistant_exec.py` 后运行） |
| `sandbox_kill` action | 终止当前沙箱进程 |
| `sandbox_status` action | 返回状态 + 最近 20 行输出 |
| `GET /api/sandbox-output` | SSE 实时流式推送沙箱 stdout（同 pipeline-output 模式） |
| `GET /api/sandbox-status` | REST 查询沙箱状态 + 最近 50 行缓存 |
| chat context 扩展 | 每次对话自动注入沙箱状态 + 可用脚本列表 |
| system prompt 更新 | LLM 知道所有沙箱工具及其用法 |

**使用方式**：在 Assistant 标签页对话，如「帮我运行 solver.py 看看输出」，LLM 会自动生成 `<action>{"type":"sandbox_run","script":"solver.py"}</action>`，前端执行后可监听 `http://localhost:8501/api/sandbox-output` 获取实时输出流。
